# 大方針Q：支払い申告メモ機能

確定日: 2026-05-26
GitHub Issue: #13

## 目的
会計係が2人いる状況で「渡した/受け取っていない」のトラブルを防止するため、
支払い申告時に証拠記録（日付・場所・受取人）を残す。

## 中方針

| # | 内容 |
|---|------|
| 中Q-1 | DB設計（payment_statusesにメモ関連カラム追加） |
| 中Q-2 | 申告UI（支払い申告時の入力フォーム） |
| 中Q-3 | メモ閲覧（本人＋権限者のみ表示） |

## 小方針

| # | 親 | 内容 | 詳細 |
|---|-----|------|------|
| 小Q-1 | 中Q-1 | payment_statusesに3カラム追加 | claim_date(date), claim_place(text), claim_recipient(text) |
| 小Q-2 | 中Q-1 | RLS設定 | メモ閲覧は本人 + グループのleader/moderatorのみ |
| 小Q-3 | 中Q-2 | 「支払った」ボタンをフォームに変更 | 日付(カレンダーピッカー), 場所(自由テキスト), 受取人(自由テキスト) 全項目必須 |
| 小Q-4 | 中Q-3 | 承認画面にメモ表示 | approve-list内で申告メモを表示 |
| 小Q-5 | 中Q-3 | イベント詳細画面で本人にメモ表示 | 申告済みの場合、自分のメモを確認可能 |

## 閲覧権限

- 申告者本人 → 自分のメモのみ閲覧可
- leader / moderator → 全員のメモを閲覧可
- 一般メンバー → 他人のメモは閲覧不可

## 依存前提
- payment_statusesテーブルが既存であること
- RLSが有効であること
